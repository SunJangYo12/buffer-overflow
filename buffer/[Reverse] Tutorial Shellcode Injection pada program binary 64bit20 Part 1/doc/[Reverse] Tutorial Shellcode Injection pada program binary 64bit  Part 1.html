[Reverse] Tutorial Shellcode Injection pada program binary 64bit: Part 1

Habibie Faried
Jan 22, 2017 · 12 min read

Halo semua! sesuai dengan janji pada artikel pengantar saya mengenai reverse. Kali ini kita langsung coba praktek bagaimana cara melakukan shellcode injection pada program binary target.

[!] Sebelum lanjut, kamu harus sudah punya basic knowledge mengenai assembly. Minimal telah membaca artikel perdana saya pada tag “reverse”.

Dan ini mungkin akan menjadi sangat sulit, pilih antara kamu mau membaca sampai habis & paham atau tidak membaca sama sekali.

Setup Program

Silahkan ketikkan program C berikut ini

#include <stdio.h>
#include <unistd.h>
int vuln() {
    char buf[80];
    int r;
    r = read(0, buf, 400);
    printf("\nMembaca %d bytes. isinya adalah %s\n", r, buf);
    puts("Semuanya oke!");
    return 0;
}
int main(int argc, char *argv[]) {
    setuid(0);
    setgid(0);
    printf("Welcome to my simple program");
    vuln();
    return 0;
}

Memahami crashnya program

Kamu (dan saya) tentunya bertanya-tanya mengapa program ini crash ketika dimasukkan string sepanjang 400 karakter? Jawaban yang kita punya sementara adalah karena kita telah menghapus suatu alamat, dimana alamat ini sangat penting bagi instruction pointer untuk menjalankan alamat program selanjutnya. Hal ini dikarenakan kita hanya memesan 80 bytes of stack, kalau dimasukkan 400 akan menjadi SANGAT MUNGKIN terjadi overwriting pada alamat lainnya.

   0x000000000040068e <+72>: leaveq 
   0x000000000040068f <+73>: retq
Program received signal SIGSEGV, Segmentation fault.
0x000000000040068f in vuln ()

Tebakan mudah saya, alamat yang kita hancurkan adalah alamat stack pointer, itulah mengapa gdb menunjukkan error pada alamat 0x40068f, alamat tersebut memiliki fungsi ret yaitu stack pointer seharusnya akan direset dan menunjuk ke alamat lain sesuai dengan instruksi $rip (Instruction Pointer)

Menentukan letak Stack Pointer (sp)

Hal yang pertama wajib dilakukan adalah kamu WAJIB tahu adalah mengetahui lokasi Stack Pointer (rsp) ketika terjadi error.

Intinya adalah mengganti alamat fungsi yang seharusnya dipanggil oleh sistem ke alamat fungsi milik kamu (shellcode)

Sekarang, mari kita bedah satu per satu line of code dari assembly yang ada

push   %rbp

Register %rbp masuk kedalam stack frame/fungsi vuln. Register ini memuat alamat dari base pointer fungsi vuln

mov    %rsp,%rbp

Value dari register base pointer tersebut dicopykan ke register stack pointer (%rsp). Stack Pointer inilah yang akan jalan-jalan di fungsi/stack frame vuln()

sub    $0x60,%rsp

Stack pointer memesan 0x60 (80) bytes. Sehingga posisi stack pointer ini ada diatas

Harus diingat bahwa semakin keatas, maka semakin kecil alamat memory yang dimiliki. Lihat gambar di bawah ini

lea    -0x60(%rbp),%rax
mov    $0x0,%edi
mov    $0x190,%edx
callq  0x400500 <read@plt>

Saya juga tidak terlalu mengerti ketika sudah mau masuk fungsi read. Namun, saya mengambil kesimpulan mudah bahwa register %rax, %edi dan %edx merupakan register yang valuenya akan menjadi parameter fungsi read. Dengan masing-masing bernilai 0x60 (80), 0x0 (0) dan (0x190) 400.

Silahkan kamu coba baca & pelajari sendiri baris selanjutnya :D

Dan setelah itu, program akan keluar dengan memanggil leave dan ret.

Mungkin disinilah letak masalah itu muncul, alamat yang ditunjuk setelah keluar dari stack frame vuln() itu tidak ada.

Exploiting the Bug

Datangnya bus error?

Kenapa yang bagian “C” itu dihapus?

Sebenernya ga ngaruh sih gan, karena biar si program shellcode saya bersih aja dari karakter “C” gajelas itu. Okeh saya modif program python menjadi seperti ini
